Multiplayer Client/Server interface

Client to Server

Note: the entity is an id assigned to the player when he tries to register. obj and plyr sent through the udp sockets are packed representations of an object and a player respectively.

Supported commands and when the should be sent:

when the client presses a button

(entity) keypressed (button)

when the client releases a button

(entity) keyreleased (button)

when the client requests an update of level level_name Note: it is expected that the server responds with updatePlayer and updateObject

(entity) update (levelName)

when the client registers a character by the name of name with costume costume 

(entity) register (name) (costume)

TODO: registration response with entity assignment

when the client decides to enter a specific level

(entity) enter (level)

TODO: character costume switch

Server to Client

when the server gets a request from an entity to update this is sent to a single client

(entity) updatePlayer (plyr)

when the server gets a request from an entity to update
this is sent to a single client

(entity) updateObject (obj)

when the server notices a player switching between backend gamestates this is sent to all clients

(entity) stateSwitch (fromLevel) (toLevel)

when the server wants to tell a client to make a sound this may or may not be sent to all clients

(entity) sound (soundFile)

Communication Interface for Multiplayer Client/Server

Note: the entity is an id assigned to the player when he tries to register. obj and plyr sent through the UDP sockets are packed representations of an object and a player respectively.

Packed Representation of Nodes

The UDP sockets we've implemented do not allow sending arbitrary tables. To that end, we needed to incorporate a straightforward mechanism as follows:

We design these packed representations of nodes to contain only those attributes which the client needs to draw it.

Format

The string that represents the node is a sequence of parameters, each of which end with ?

A parameter is a triplet separated by two ! s

The triplet is identified by TYPE,NAME, and VALUE

TYPE

TYPE is an element of the set {B,N,S}

B indicate the parameter is a boolean

N indicates the parameter is a number

S indicates the parameter is a string

NAME

NAME is the name of the parameter

VALUE

VALUE is the value of the parameter

Example:

S!id!player1729?N!x!100?N!y!200?

translates to

{id="player1729", x=100, y=200}
